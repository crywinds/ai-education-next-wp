# 如何移动项目到其他位置

## 概述

项目文件夹现在可以移动到主机的任何位置，所有批处理文件都会自动适配新的路径。

## 移动步骤

### 1. 停止所有正在运行的服务
- 关闭所有相关的命令行窗口
- 确保端口 5200-5299 没有被占用

### 2. 移动项目文件夹
- 将整个项目文件夹（`new-next-wp`）移动到您想要的新位置
- 例如：从 `F:\FIFA\new-next-wp` 移动到 `D:\Projects\new-next-wp`

### 3. 验证文件完整性
移动后，确保以下文件存在：
- `start.bat` - 启动前端开发服务器
- `start-all.bat` - 启动所有服务
- `package.json` - Node.js 项目配置
- `node_modules/` - 依赖包（可选，可以重新安装）

### 4. 重新安装依赖（可选）
如果移动后遇到问题，可以在新位置重新安装依赖：
```batch
npm install
```

### 5. 运行脚本
移动后，直接双击 `start.bat` 或 `start-all.bat` 即可启动项目。

## 技术说明

批处理文件使用了以下技术来自动适配路径：

1. **`%~dp0`** - 获取批处理文件所在的目录路径
2. **`cd /d "%~dp0"`** - 切换到批处理文件所在的目录
3. **动态路径显示** - 显示当前项目的实际路径

## 注意事项

1. ⚠️ **保持文件夹结构完整**：移动时请确保移动整个项目文件夹，不要遗漏任何文件
2. ⚠️ **环境变量**：如果项目使用了 `.env` 或 `.env.local` 文件，请确保这些文件也被移动
3. ⚠️ **Node.js 版本**：确保新位置的 Node.js 版本与项目要求一致
4. ⚠️ **数据库连接**：如果使用本地数据库，确保数据库服务在新位置仍然可以访问

## 常见问题

### Q: 移动后脚本无法运行？
A: 检查是否使用了正确的编码（GBK/GB2312），或者尝试右键点击批处理文件，选择"编辑"查看内容是否正确。

### Q: 移动后端口仍然被占用？
A: 脚本会自动检测并尝试关闭占用端口的进程。如果无法自动关闭，请手动关闭相关进程。

### Q: 移动后依赖包丢失？
A: 运行 `npm install` 重新安装依赖包即可。
